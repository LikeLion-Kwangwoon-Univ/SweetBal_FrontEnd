# [태그이름] 커밋내용 (#이슈번호)
# 커밋 내용은 명사로 끝나며 마침표를 사용하지 않는다.
# 예) [Feat] 회원 가입 기능 구현 (#12)

################
# [Feat]
# 새로운 기능 구현
#
# [Fix]
# 버그, 오류 수정
#
# [Hotfix]
# issue나 QA에서 급한 버그 수정
#
# [Docs]
# 문서 수정
#
# [Test]
# 테스트 코드 추가 및 업데이트
#
# [Chore]
# 코드 수정, 내부 파일 수정
#
# [Del]
# 불필요한 코드 삭제
#
# [Refactor]
# 기능은 바꾸지 않되, 코드의 구조를 변경
#
# [Merge]
# 다른 브랜치를 merge 할 때 사용
#
# [Add]
# Feat 이외의 부수적인 코드 추가, 라이브러리 추가, 새로운 파일 생성 시
#
# [Rename]
# 파일 이름 변경
#
# [Move]
# 프로젝트 내 파일이나 코드의 이동
